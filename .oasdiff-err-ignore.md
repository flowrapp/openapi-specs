# oasdiff Breaking Changes — Ignored Errors

These breaking changes are false positives caused by the OAS 3.0 → 3.1 nullable syntax
migration (`nullable: true` → `type: ['string', 'null']`). oasdiff's stable version does
not yet recognise that these are semantically equivalent (see oasdiff/oasdiff OPENAPI-31.md).

This file can be removed once the migration commit is merged into the base branch,
because subsequent diffs will compare identical nullable syntax on both sides.
